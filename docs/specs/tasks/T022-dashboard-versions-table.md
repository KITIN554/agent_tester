# T022 — Сравнительная таблица v0/v1/v2 в dashboard

## Приоритет: Важно (#7, ~1 час)

## Цель
Дать быстрый ответ на вопрос «дала ли наша работа прирост»: таблица
RQS/PQS/RS/SS/ES для трёх ключевых прогонов (v0 = baseline до правок,
v1 = после T016 prompts, v2 = после T019 evolve cycle) с подсветкой
улучшений и регрессий.

## Что нужно сделать

1. Новая вкладка `📈 Versions` (после «Сводка», перед «Динамика»).
2. `dashboard/views/versions.py`:
   - В sidebar — три селекта: «v0 / v1 / v2» (опции — все run_id корзины).
   - Дефолты: v0 = первый по времени; v1 = средний; v2 = последний.
   - Таблица: строки — RQS, PQS, RS, SS, ES, total_cost_usd, pass_rate;
     столбцы — v0, v1, v2, Δ(v1-v0), Δ(v2-v1).
   - Цвет ячейки дельты: зелёный +, красный −, серый 0.
   - Под таблицей — миниграф Δ_RQS по версиям через plotly bar.
3. Опционально: версионные тэги через `tester baseline --tag v1 …`
   (сохранять в `reports/runs/.tags.json`). Можно отложить — для
   первой итерации хватит ручного выбора в селектах.

## Acceptance criteria

- [ ] `dashboard/views/versions.py` рендерит таблицу + миниграф.
- [ ] `dashboard/app.py` подключает вкладку.
- [ ] Если в корзине <3 прогонов — сообщение, без падения.
- [ ] Закоммичено: `feat(dashboard): version comparison table (v0/v1/v2)`
