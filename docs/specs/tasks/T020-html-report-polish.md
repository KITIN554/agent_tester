# T020 — Улучшить HTML-отчёт (CSS, progress bars, иконки)

## Приоритет: Важно (#5, ~1.5 часа)

## Цель
Сделать HTML-отчёт пригодным для человека: видеть статус с одного взгляда,
понимать просадки без сравнения чисел в голове, листать длинные трассы.

## Что нужно сделать

1. **Прогресс-бары для метрик** — каждая ячейка значения метрики
   показывает заполненность относительно порога:
   ```html
   <div class="bar"><div class="fill" style="width: 87%"></div></div>
   ```
   Цвет: зелёный если ≥ порога, жёлтый при −5%, красный при просадке больше.
2. **Иконки статусов** (Unicode, без внешних шрифтов):
   - ✅ pass / ✓
   - ⚠️ partial / fail-non-critical
   - ❌ fail / block
3. **Свёртывание длинных трасс** — для multi_turn (10+ шагов) показывать
   только TOOL_CALL/TOOL_RESULT по умолчанию, диалоговые шаги под `<details>`.
4. **Sticky-навигация** — якоря «К gate / К метрикам / К провалам»;
   фиксированная top-bar с gate-биркой.
5. **Цвет-кодинг рубрик в таблице failures** — фон ячейки рубрики
   по статусу.
6. **Адаптация CSS** для печати (`@media print`).

## Acceptance criteria

- [ ] `tester/templates/report.html.j2` содержит progress-bar макрос.
- [ ] `tester/templates/style.css` добавляет стили для bar/fill/иконок,
  `@media print` секцию.
- [ ] Существующие тесты `tests/test_reporter.py` зелёные; добавить тест
  «прогресс-бар присутствует в HTML».
- [ ] Откомпилированный `index.html` любого реального прогона открывается
  и визуально читается без CSS-багов.
- [ ] Закоммичено: `feat(report): polish HTML — progress bars, icons, foldable traces`
